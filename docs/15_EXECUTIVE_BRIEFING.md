# 15_EXECUTIVE_BRIEFING.md

## Project one-liner

`SymbiontOS` is a decision-first workforce control plane that helps an AI transformation lead route knowledge work to a Human, an Agent, or a Hybrid team with visible governance and audit evidence.

## Market friction

Teams are adopting AI agents faster than their operating systems can govern them. Leaders need to decide which tasks are safe for agents, which tasks need Human accountability, and which tasks need a Hybrid workflow, but those decisions are often manual, inconsistent, and hard to audit.

## Target cohort

The first user is an Innovation / AI transformation lead responsible for introducing agents into real work without losing control, trust, or explainability.

## Why now

Agent tools are becoming useful enough to enter everyday knowledge work, but enterprise adoption is blocked by practical questions: where should agents be used, who approves them, what happens when the output is risky, and how is the decision recorded?

## Product wedge

The wedge is not another chat interface or broad workflow suite. It is the control layer before agentic work scales:

`task intake -> Human / Agent / Hybrid recommendation -> governance -> controlled execution -> output review -> audit trail`

## Core workflow

1. Submit or choose a knowledge-work task.
2. Analyze task clarity, judgment, sensitivity, risk, urgency, and budget pressure.
3. Recommend Human, Agent, or Hybrid execution.
4. Explain the recommendation and alternatives.
5. Apply governance: Approved for launch, Needs human review, or Blocked.
6. Select an eligible curated execution option.
7. Launch allowed work or pause for Human review.
8. Run controlled local Agent output when allowed.
9. Record a Human output review decision.
10. Show lifecycle and audit trail evidence.

## Architecture summary

The current MVP is a React + Vite frontend. Product behavior is generated by deterministic JavaScript modules under `app/src/logic`, with `buildTaskFlow(...)` acting as the central assembly line for recommendation, governance, marketplace selection, execution state, lifecycle, and audit trail.

Built-in scenarios are source-controlled. Local demo state uses browser `localStorage` only. The app works without secrets, network access, or external model calls. Phase 19B adds an optional live AI draft adapter that can call a provider only when the user enters a session-only API key in the UI. No model controls routing, governance, blocked status, lifecycle policy, or audit policy.

## Current MVP boundary

The current repository proves the frontend workflow and control-plane model. It does not claim production readiness. It is intentionally scoped to:

- task intake
- deterministic recommendation
- explainability
- governance checks
- curated sample execution options
- Human review controls
- controlled local Agent Runner
- optional live AI draft mode as a pluggable execution source
- Agent output review
- lifecycle and audit evidence
- local browser persistence for demo state

## Business value

SymbiontOS gives leaders a practical way to introduce agents safely. It helps speed up low-risk work, keeps Humans accountable for judgment-heavy work, and creates a repeatable review path for Hybrid work.

## Technical value

The repo separates important product concepts into inspectable modules: task analysis, recommendation, governance, marketplace selection, Human review, execution tracking, Agent run output, output review, lifecycle, and audit trail. That separation makes the demo easier to validate now and easier to replace with production infrastructure later.

## Why this fits Founder Mode

The project starts from a narrow operating pain instead of trying to become a large enterprise platform. It proves one sharp wedge: trusted routing and governance for agentic knowledge work. The current repo shows fast execution, visible scope control, and a path from MVP evidence to production infrastructure.

## What is real now

- A working frontend MVP in `app/`.
- Built-in scenarios for Agent, Hybrid, Human, and Blocked paths.
- Deterministic scenario validation with `npm.cmd --prefix app run validate:scenarios`.
- Browser-local custom tasks and review decisions.
- Controlled local Agent Runner output after governance allows launch.
- Optional live AI draft output after governance allows launch and the user provides a session key.
- Human output review decisions saved in browser local state.
- Lifecycle and audit trail evidence visible in Task Detail.
- Documentation for architecture, domain model, demo scripts, deployment, QA, and submission.

## What comes next

Next production work should add durable infrastructure behind the same workflow:

- backend API routes for tasks, recommendations, governance, reviews, executions, Agent runs, and audit events
- database tables for shared state and review history
- queue jobs for long-running Agent execution
- provider adapters for real model or agent calls behind server-side credentials
- authentication and role-based approval permissions
- observability for recommendation quality, policy blocks, provider latency, failures, and audit integrity
